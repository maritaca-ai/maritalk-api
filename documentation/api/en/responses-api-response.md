# The response object

Represents a response returned by the model in the Responses API, based on the provided input.

<div class="container">

<div class="container-esquerda">

### id `string`

Unique response identifier (e.g., `resp-abc123def456`).

---

### object `string`

The object type, always `"response"`.

---

### created_at `integer`

Unix timestamp (in seconds) of the response creation.

---

### status `string`

Response status: `"completed"`, `"failed"`, or `"in_progress"`.

---

### model `string`

The model used to generate the response.

---

### output `array`

List of output items. Each item can be a message or a function call.

<details>
<summary>message (object)</summary>

- **id** (string): Unique message identifier.
- **type** (string): Always `"message"`.
- **role** (string): Always `"assistant"`.
- **status** (string): `"completed"` or `"in_progress"`.
- **content** (array): List of content parts:
  - **type** (string): `"output_text"`.
  - **text** (string): Text generated by the model.
  - **annotations** (array): Annotations (usually empty).

</details>

<details>
<summary>function_call (object)</summary>

- **id** (string): Unique item identifier.
- **type** (string): Always `"function_call"`.
- **name** (string): Name of the function to call.
- **arguments** (string): Arguments as serialized JSON.
- **call_id** (string): Call identifier (used to send the result back).
- **status** (string): `"completed"`.

</details>

---

### usage `object`

Token usage statistics.

<details>
<summary>Show properties</summary>

- **input_tokens** (integer): Tokens consumed in the input.
- **output_tokens** (integer): Tokens generated in the output.
- **total_tokens** (integer): Total tokens (input + output).
- **input_tokens_details** (object or null):
  - **cached_tokens** (integer): Tokens from cache.
- **output_tokens_details** (object or null):
  - **reasoning_tokens** (integer): Tokens used for reasoning.

</details>

---

### error `object or null`

Error object when `status` is `"failed"`.

<details>
<summary>Show properties</summary>

- **code** (string): Error code (e.g., `"server_error"`, `"internal_error"`).
- **message** (string): Descriptive error message.

</details>

---

### incomplete_details `object or null`

Details about why the response is incomplete (e.g., reached `max_output_tokens`).

<details>
<summary>Show properties</summary>

- **reason** (string): Interruption reason (e.g., `"max_output_tokens"`).

</details>

---

### instructions `string or null`

System instructions sent in the request.

---

### temperature `float or null`

Sampling temperature used.

---

### top_p `float or null`

Top-p used.

---

### max_output_tokens `int or null`

Configured token limit.

---

### metadata `object`

Request metadata (key-value pairs).

---

### tools `array`

Tools that were available to the model.

---

### tool_choice `string or object or null`

Tool usage configuration.

---

### parallel_tool_calls `boolean`

Whether parallel tool calls were enabled.

---

### store `boolean`

Whether the response was stored.

---

## Streaming Events

When `stream: true`, the API returns events via Server-Sent Events (SSE).

| Event | Description |
|---|---|
| `response.created` | Response created |
| `response.in_progress` | Generation started |
| `response.output_item.added` | New output item added |
| `response.content_part.added` | New content part started |
| `response.output_text.delta` | Text fragment generated |
| `response.output_text.done` | Full text finalized |
| `response.content_part.done` | Content part finalized |
| `response.output_item.done` | Output item finalized |
| `response.completed` | Response complete (with `usage`) |
| `response.failed` | Error during generation |

All events include a `sequence_number` field incremented with each event.

---

## HTTP Error Codes

| Status | Description |
|---|---|
| `200` | Success |
| `400` | Unsupported parameter or invalid request |
| `401` | Missing or invalid API key |
| `422` | Validation error |
| `429` | Rate limit exceeded |
| `500` | Internal server error |
